JavaScript: High-level Overview

JavaScript is a high-level, Object-oriented, Multi-paradigm Programming Language

JavaScript is :

  • High-level :In Low level languages (e.g. C), Developer has to manage resources manually. But in high-level languages(e.g. JS, Python), the developer doesnot have to worry about how to manage resources. JavaScript does it automatically.
  • Garbage-collected Here JS removes old(unused) objects from memory automatically. So the developer don't have to manually clean up the memory.
  • Interpreted/JIT Compiled:JS codes are converted into machine code entirely(with out storing portable files like c++ ) at once and then execute immediately.
  • Multi-paradigm approach of structuring code. Js supports Procedural Programming(Linear programming), OOP and Functional Programming.
  • prototype-based OO Everything in JS is an object except primitive types. there is a prototype (Blueprint) that contains methods of objects. So when we write objects, it inherits the methods from the blueprint(prototype)
  • First-class functions In a language of First-class functions, functions are simply treated as variables. We can pass them into other functions, and return from functions. Example of passing function into another function as argument: p.addEventlistener("click", displayMessage)
  • Dynamically-typed language No data type definitions. It is deterrmined at runtime. Data type is automatically changed(let x=="NN" x = 10).
  • Single-threaded and Non-blocking event loop Javascript uses event loop to handle concurrent tasks(multiple tasks happening at the sae time). Event loop takes long long runnig tasks, executes them in background, and puts them back in the main thread once they're finished.

Interpreted vs Compiled vs JIT-compiled

  • Compiled here the Entire code is converted into machine code at once. Then it keeps portable file and finally executes it.
  • Interpreted here code is converted to machine code line by line. It is much slower than compiled languages.
  • JIT Compiled here Entire code is converted into machine code (with out storing portable files like c++ ) at once and then execute immediately.

JavaScript Engine

JavaScript Engine is a program that execute JavaScript code.

E.g. V8 Engine

Has 2 parts: Callstack and Heap

Callstack : is where our codes are executed.

Heap: is where objects are stored.

Modern JIT Compilation step of JavaScript

  1. Parsing: first the code is parsed into a data structure called AST(Abstract Syntax Structure or tree). Also checks if there is syntax error.
  2. Compilation the code will be converted to machine code with JIT.
  3. Execution the code will be executed. It happens in call Stack.
  4. Optimization strategy JavaScript first creates a very unoptimized version of machine code in the beginning to start executing as fast as possible. In the background, this code is optimized and recompiled during the already running prgram execution. It can be done multiple times.

    5. After each optimization, the unoptimized code is simply swapped by the optimized one without ever stopping execution. It makes modern engines (V8) fast.

JavaScript Runtime

is a container including all the things that we need to use JavaScript(e.g. Browser). Without engine, there is no runtime.

JavaScript runtime includes Engine, Web API and Callback Queue.

JavaScript Engine: is a program that executed JavaScript code.

Web API : is functionality provied to the engine. It is outside of the browser but JS Engine access it on window object. It includes DOM, Fetch API, Timers...

CallBack queue data structure containing all callback functions. Example: callback function from DOM event listener. When user clicks on a button, the callback function is put on callback queue in JS Engine. Then when the callstack is empty, the callback function is passed to the callstack so it can be executed. These all are handled by event loop. Basically, event loop takes callback functions from callback queue, and puts it to callstack for execution./p>

Execution Contexts and the call stack

Steps:

  • First it is compiled so it becomes ready to be executed
  • Creation of global execution content for top-level code Top-level code is global or which is outside function. e.g const name = "Natty" declared globally. Execution context is an environment in which a piece of javascript is executed. It stores all the necessary information for some code to be executed. There is exactly one/default global execution content(EC) top-level code.
  • Execution of top-level code (inside global EC)
  • Execution of functions and waiting for callbacks For each function call, one new execution context is created.
  • The all together make the call stack

What's inside execution context

  1. Variable environment: - let, const, var declarations , -functions, -argument object
  2. Scope chainreferences to variables located outside the function
  3. this keyword

    4. Argument object and this keyword can't be in arrow functions

Call Stack explained

It is where our JavaScript codes are executed in javascript engine. When there are multiple functions(functions calling another functions), Execution contexts get stacked on top of each other, to keep track of where we are in the execution and which to execute next. The one on the top of the stack is the one currently running. Once it finishes execution, it will be out of the callstack and the previous will be execued. In JavaScript callstack, order of execution willnot be lost.
